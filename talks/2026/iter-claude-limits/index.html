Part 0: Introductory Concepts

Understanding the building blocks of the Claude ecosystem

Prompts

Ephemeral, conversational instructions you provide in natural language during a conversation.

  • Use for: One-off requests, conversational refinement, immediate context, ad-hoc instructions.
  • Don't persist across conversations. For repeated workflows, capture as Skills or project instructions.
  • Pro-tip: If you find yourself typing the same prompt repeatedly → transform it into a Skill.

Projects

Self-contained workspaces with their own chat histories and 200K context window knowledge bases.

  • Everything uploaded becomes available across all chats within the project.
  • RAG mode expands capacity by 10x when approaching context limits.
  • Use for: Persistent context, workspace organization, team collaboration, custom instructions.
  • Signal to use Skills instead: Copying same instructions across multiple Projects.

Commands

Custom slash commands that trigger specific workflows or actions when invoked.

  • Purpose: Execute predefined operations like /project:review or /deploy.
  • Example: A markdown file that runs git diff and prompts Claude to review for security/performance issues.
  • Location: Defined in .claude/commands/ folder or global ~/.claude/commands/.
  • Use for: Repetitive workflows that need manual triggering but consistent execution.

Rules

Path-scoped configuration files that provide modular, file-specific instructions to Claude.

  • Purpose: Avoid bloating CLAUDE.md with rules that only apply to specific file types or directories.
  • How they work: Rules only load when editing files matching specified path patterns.
  • Example: API design rules for src/api/**/*.ts. Handlers must return { data, error } shape.
  • Location: .claude/rules/*.md with YAML frontmatter defining paths.

Skills

Specialized training manuals containing instructions, scripts, and resources that Claude discovers and loads dynamically when relevant.

  • How they work: Progressive disclosure. Metadata loads first (~100 tokens), full instructions when needed (<5k tokens), bundled files only as required.
  • Use for: Organizational workflows (brand guidelines, compliance), Domain expertise (Excel, PDF manipulation), Personal preferences (coding patterns).
  • Signal to create: Typing the same prompt repeatedly across conversations.

Subagents

Specialized AI assistants with their own context windows, custom system prompts, and specific tool permissions.

  • Handle discrete tasks independently and return results to the main agent.
  • Claude automatically delegates based on subagent descriptions, or you can explicitly request one.
  • Use for: Task specialization (code review, security audits), context management, parallel processing, tool restriction.
  • Subagents can use Skills for portable expertise.

MCP

Model Context Protocol. An open standard for connecting AI assistants to external systems where data lives.

  • Purpose: Universal connection layer between Claude and tools/data sources (Google Drive, Slack, GitHub, databases).
  • How it works: MCP servers expose data/capabilities; MCP clients (like Claude) connect to them.
  • Use MCP for: Access to external data and business tools.
  • Use Skills for: How to use those tools or follow procedures.

Part 1: Anatomy of .claude/

The control center for agentic behavior in your project.

The Two Domains

your-repo/.claude/

Project-level

  • Committed to Git.
  • Shared team configuration.
  • Project-specific rules, agents, and slash commands.

~/.claude/

Global-level

  • Lives on your local machine.
  • Personal preferences & overrides.
  • Global memory and cross-project skills.

CLAUDE.md: The Instruction Manual

  • Loaded directly into the system prompt. Dictates behavior.
  • Keep it under 200 lines to preserve context and instruction adherence.
  • DO include: Run commands (npm run test), architecture (Express/Prisma), strict conventions (use zod, no console.log).
  • DON'T include: Linter configs, long theoretical explanations.
  • Pro-tip: Use CLAUDE.local.md for uncommitted personal overrides.

Scaling Rules: .claude/rules/

Avoid a bloated CLAUDE.md by using modular rules scoped by file path.

--- paths: - "src/api/**/*.ts" - "src/handlers/**/*.ts" --- # API Design Rules - All handlers return { data, error } shape - Never expose internal error details to clients

Claude only loads this context when editing matching files.

Agentic Extension: Commands & Skills

  • Commands (/commands/): Custom slash commands (e.g., /project:review).
    Example: A markdown file that runs git diff and prompts Claude to review for security/performance.
  • Skills (/skills/): Workflows Claude invokes autonomously based on task matching.
    Includes a SKILL.md describing when to trigger it (e.g., "Use when reviewing PRs"). Can bundle supporting docs (like DETAILED_GUIDE.md).

Sub-agents & Permissions

Agents (/agents/)

  • Specialized personas with isolated contexts.
  • Define tool access & model (e.g., a "Code Reviewer" agent restricted to Haiku and Read-only tools).

Settings (settings.json)

  • Controls what Claude is allowed to do.
  • Allow List: Bash(npm run *), Read, Write
  • Deny List: Bash(rm -rf *), Read(./.env)
  • Provides an essential safety net for headless/agentic runs.

Part 2: The Token Game

"Claude doesn't count messages. It counts tokens."

1. The Core Paradigm Shift

  • The Problem: Every subsequent message forces Claude to re-read the entire history.
  • Message 30 costs 31x more tokens than Message 1!
  • The Fix: Never send follow-up corrections (e.g., "No, I meant X").
  • Instead: Edit your original prompt → Fix it → Regenerate.
  • Rule: Fix the prompt, don't feed the history.

2. Context Window Management

Fresh Chats

  • Start a new chat every 15–20 messages.
  • In long chats, 98.5% of tokens are wasted on re-reading history.
  • Workflow: Ask Claude to summarize → Start new chat → Paste summary.

Batch Prompting

  • Three separate prompts = Three context loads.
  • One prompt with 3 tasks = One context load.
  • Provides better output because the LLM sees the full picture instantly.

3. Optimize Features & Memory

  • Projects (Context Caching): Upload recurring files (PDFs, docs) to a Project. Claude caches it; it won't re-tokenize on every conversation.
  • Memory & Preferences: Save your role, style, and rules in Settings → Memory. Stops you from wasting tokens on "Act as a..." boilerplates.
  • Disable Unused Tools: Turn off Web Search, Connectors, and "Advanced Thinking" by default. They eat tokens invisibly.

4. Model Routing & Time Management

  • Model Tiering:
    • Haiku: Drafts, grammar, formatting (Saves 50-70% budget).
    • Sonnet: Real work, coding, medium cost.
    • Opus: Deep thinking, architecture, high cost.
  • Rolling Window: Limits operate on a 5-hour rolling window. Spread work into morning/afternoon/late night sessions.
  • Peak Hours: Avoid heavy tasks during 5:30 PM–11:30 PM IST (limits drain faster).

Summary: The Golden Rules

  • Treat Tokens like Currency: Edit prompts, batch tasks, and start fresh chats to preserve your budget and context window.
  • Right-size your Models: Default to Haiku, escalate to Sonnet/Opus only when required.
  • Infrastructure as Prompt: Treat .claude/ as code. Keep CLAUDE.md dense and leverage path-scoped /rules/ for scalability.
  • Guardrails: Always configure settings.json to deny destructive commands and protect secrets.
